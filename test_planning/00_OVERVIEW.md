# Test Planning Overview

This directory contains the testing strategy for the ColdCake project.
The application has been divided into 7 phases for systematic verification.

## Phases
1. **[Setup & Auth](./PHASE_1_SETUP_AUTH.md)**: Environment, DB connectivity, and User Authentication.
2. **[Product & Inventory](./PHASE_2_PRODUCT_INVENTORY.md)**: Core catalog and stock management.
3. **[Sales & Orders](./PHASE_3_SALES_ORDERS.md)**: E-commerce flows, cart, checkout, invoicing.
4. **[CRM & Messaging](./PHASE_4_CRM_MESSAGING.md)**: Customer profiles, debts, and communication.
5. **[Special Features](./PHASE_5_SPECIAL_FEATURES.md)**: Gold prices, Maps Scraper, Blog.
6. **[Admin & Settings](./PHASE_6_ADMIN_REPORTS.md)**: Dashboard, configurations, reports.
7. **[E2E & Final](./PHASE_7_E2E_FINAL.md)**: Full user journeys, performance, security.

## How to use
Start with Phase 1. Do not proceed to subsequent phases until the critical checkboxes in the current phase are marked as `[x]`. 
Each phase includes "Improvements & Suggestions" which should be considered for the roadmap.
